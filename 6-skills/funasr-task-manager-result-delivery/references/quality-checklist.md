# 结果质量检查规则

## 检查层级

### 1. 非空检查（必须）

- 转写文本不为空字符串
- 转写文本不为 `null`
- 文本长度 > 0

**空文本处理**：标记异常，建议检查：
- 音频是否为静音
- 语言设置是否与实际语音匹配
- 音频质量（信噪比）是否足够

### 2. 乱码检查

- 文本中不应出现大量不可识别字符
- 中文语音应包含中文字符
- 连续特殊字符超过 5 个视为可疑

**乱码处理**：标记异常，建议检查：
- 音频编码是否正确
- 输入格式是否与实际内容匹配
- 是否需要不同的语言模型

### 3. 过短文本检查

- 音频时长 > 30 秒但文本 < 10 字 → 可疑
- 音频时长 > 60 秒但文本 < 20 字 → 可疑

**过短处理**：提醒可能是：
- 大段静音
- 背景噪声
- 音频截断

### 4. 批量一致性检查

- 成功任务数 + 失败任务数 = 总任务数
- 没有"状态未知"的遗漏任务
- 所有成功任务均有非空结果

## 质量评级

| 评级 | 条件 |
|------|------|
| ✅ 正常 | 文本非空、无乱码、长度与时长匹配 |
| ⚠ 警告 | 文本过短或可疑字符比例高 |
| ❌ 异常 | 文本为空或完全乱码 |

## 批量汇总格式

```
质量报告:
  ✅ 正常: {normal_count}/{total}
  ⚠ 警告: {warning_count}/{total}
  ❌ 异常: {error_count}/{total}

  异常文件:
  - {filename}: 空文本
  - {filename}: 文本过短（{text_length} 字 / {duration}s 音频）
```
