# Skills 评审报告与行动方案

日期：2026-04-02

## 评审范围

对 `6-skills/` 目录下的两个 Agent Skill 进行完备性检查，包括 CLI 指令覆盖度、参数设计、SKILL.md 文档质量、README 介绍情况，以及是否需要新增其他 Skill。

## 一、reset-asr-db-before-test

### CLI 参数覆盖度

| 参数 | 覆盖的任务分类 | 评价 |
|------|--------------|------|
| `--dry-run` | 只读评估 | 完备 |
| `--no-backup` | 加速/CI 场景 | 完备 |
| `--reset-servers` | 服务器配置管理 | 完备 |
| `--skip-seed-servers` | 精细化服务器控制 | 完备 |
| `--clear-uploads` | 深度清理 | 完备 |
| `--force` | 非交互/CI | 完备 |

### 设计完备性

脚本功能边界清晰：仅负责 `3-dev/src/backend/data/` 下的 SQLite 状态管理。不越界干预前端、不处理 PostgreSQL 场景、不尝试管理 Docker 容器——设计约束正确。

### 发现的问题

1. **SKILL.md 未同步最新修复。** 数据库占用检测（`check_database_in_use`）、损坏数据库降级处理（`sqlite3.DatabaseError` 捕获）、WAL/SHM/journal 附属文件清理均已在脚本中实现，但 SKILL.md 的"默认行为"、"执行结果"、"安全约束"三节未体现：
   - 未提到"若数据库被其他进程占用，脚本会拒绝执行"
   - JSON 示例未展示 `servers_preservation_skipped`、`database_corrupt` 等新字段
   - dry-run 输出说明未提到损坏库时的降级行为

2. **缺少 `--verbose` 标志。** 当前 `output_result()` 只输出最终 JSON，中间过程对排障不可见。不算阻断性问题，但降低 CI 日志的可调试性。

## 二、funasr-task-manager-web-e2e

### CLI 参数覆盖度

`build_fixture_batch.py` 的参数集：

| 参数 | 覆盖的任务分类 | 评价 |
|------|--------------|------|
| `--profile` | smoke/remote-standard/standard/full | 完备，4 级分层清晰 |
| `--assets-root` | 自定义素材目录 | 完备 |
| `--output` | 自定义输出路径 | 完备 |
| `--write` | 写入默认路径 | 完备 |

### 设计完备性

流程编排完整（平台检测 → 目标选择 → fixture 构建 → 环境预检 → 用户路径模拟 → 分层断言 → 归档），参考文档齐全，跨平台适配到位。

### 发现的问题

1. **`semantic-baseline.json` 基本为空。** 两个文件条目的 `keywords_all` 和 `keywords_any` 都是空数组，语义校验层处于占位状态。`remote-standard` 有 5 个文件但 baseline 只登记了 2 个。

2. **SKILL.md 语言不一致。** 主文档是英文，但 `project-context.md` 和脚本输出是中文。在以中文为主语言的项目中增加认知切换成本。

3. **`agents/openai.yaml` 过于简略。** 仅 4 行 interface 声明，无实际指导价值。

4. **缺少 `--list` 模式。** 想快速查看当前素材目录有哪些可用文件时，只能跑 `--profile full` 看完整输出。

## 四、是否需要其他 Skills

| 能力域 | 现有覆盖 | 是否需要 Skill |
|--------|---------|---------------|
| 数据库状态管理 | `reset-asr-db-before-test` | 已有 |
| 浏览器 E2E | `funasr-task-manager-web-e2e` | 已有 |
| 后端 API/CLI 测试 | pytest 套件 + CLI `doctor` | 不需要 |
| ASR 服务器管理 | CLI `server register/probe/benchmark/update` | 不需要 |
| 性能基准测试 | CLI `server benchmark` + locustfile | 低优先级，暂不需要 |
| CI/CD 流水线 | 完全缺失 | 一次性搭建，非可重复 skill，暂不需要 |
| 数据库迁移管理 | Alembic | 标准流程，不需要 |

**结论：** 当前两个 skill 基本覆盖了 Agent 需要反复执行的自动化场景，暂不需要新增第三个 skill。
