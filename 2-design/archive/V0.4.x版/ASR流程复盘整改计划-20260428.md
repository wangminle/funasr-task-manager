## 四、根因分析总结

本次复盘揭示了两类不同性质的问题：

**技术债（可修复）**：
- 单测参数滞后于实现迭代 → 测试用例与配置不同步
- 本地 DB 未跟进迁移 → 缺少部署后自动提醒
- 解决方案：修复测试、执行迁移、AGENTS.md 记录迁移流程

**流程执行偏差（需制度约束）**：
- Skill 设计完整但 Agent 执行时"效率优先"跳过通知
- 根因是缺少强制检查清单和偏差防范文档
- 解决方案：在 Skill 中嵌入执行检查清单、偏差模式库、自检机制，在 AGENTS.md 中设立项目级规范

基于 Codex 对 v0.4.3 提交的 Code Review 反馈，修复以下 4 个 bug：

| # | 优先级 | 问题 | 修复文件 | 修复内容 |
|---|--------|------|---------|---------|
| 1 | **P1** | 飞书发送消息缺少 `receive_id_type` 参数 | `channel-file-apis.md`、`result-delivery SKILL.md` | URL 补上 `?receive_id_type=chat_id`，添加可选值说明 |
| 2 | **P2** | 飞书大文件下载无分块回退 | `channel-intake SKILL.md`、`channel-file-apis.md` | 添加 Range 分块下载代码和 234037 错误回退说明 |
| 3 | **P2** | init Phase 6/7 在环境已就绪时不可达 | `init SKILL.md` | Phase 2 和 Phase 5 退出前必须询问是否执行 Phase 6/7 |
| 4 | **P2** | Skill 安装脚本不可重复执行 | `init SKILL.md` | OpenClaw/Hermes 分支加 `rm -rf` 清旧目录；Cursor 用 `ln -sfn` |

### 影响分析

- **Bug 1 是阻断性缺陷**：飞书结果文件回传会直接失败，用户收不到转写结果
- **Bug 2 会阻断大文件场景**：>50MB 的音视频文件从飞书下载会失败
- **Bug 3 影响首次安装体验**：已部署环境无法通过 init 安装 Skills 或配置凭据
- **Bug 4 会导致静默使用旧版 Skill**：重装时旧 SKILL.md 不被覆盖，Agent 继续使用过期版本
