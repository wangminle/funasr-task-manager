# VAD 切分策略改进与评估方案

> 日期: 2026-04-25
> 前置文档: [VAD分段并行转写实施计划-20260424.md](VAD分段并行转写实施计划-20260424.md)

## 一、改进背景

阶段 0–9 已全部实现，单元测试覆盖完善。在实际长音频测试中发现以下问题：

1. **搜索窗口过窄**：当 10–11 分钟窗口内无合适静音段时直接硬切，可能切断语音
2. **没有降级策略**：硬切前未尝试寻找更短的静音段（如句间停顿 500–700ms）
3. **用户无法控制**：切分是系统强制行为，CLI/API 未提供开关参数
4. **缺少质量评估**：无法量化切分对转写质量的影响

## 二、三项改进任务

### 任务一：切分策略渐进式搜索与降级机制

**目标**：`plan_segments()` 在找不到合适断点时，逐步扩大搜索窗口并降低静音阈值，硬切作为最终兜底。

**渐进搜索策略**：

```
从当前位置 pos 开始：
第 1 轮：[pos+600s, pos+660s]  找 ≥ 0.8s 的静音，取最长
第 2 轮：[pos+660s, pos+720s]  找 ≥ 0.8s 的静音，取最长
第 3 轮：[pos+720s, pos+780s]  找 ≥ 0.8s 的静音，取最长
降级轮：[pos+600s, pos+780s]  找 ≥ 0.3s 的静音（句间停顿），取最长
硬切：  pos+780s             （最终兜底，极端情况下才触发）
```

**配置项变更** (`config.py`)：

| 配置项 | 旧值 | 新值 | 说明 |
|--------|------|------|------|
| `segment_target_duration_sec` | 600 | 600 | 不变，搜索起点 |
| `segment_max_duration_sec` | 660 | 780 | 扩展为 13 分钟硬切上限 |
| `segment_search_step_sec` | (新增) | 60 | 每轮搜索窗口宽度 |
| `segment_search_max_rounds` | (新增) | 3 | 渐进搜索最大轮数 |
| `segment_fallback_silence_sec` | (新增) | 0.3 | 降级轮最短静音阈值 |

**算法伪代码**：

```python
def _find_best_cut(pos, total, silence_ranges, target, step, rounds, min_silence, fallback_silence, maximum):
    # 第 1~N 轮：每轮搜索 [pos+lo, pos+hi]，找 >= min_silence 的最长静音
    for r in range(rounds):
        lo = target + r * step
        hi = lo + step
        candidates = [sr for sr in silence_ranges
                      if lo <= midpoint(sr) - pos <= hi and duration(sr) >= min_silence]
        if candidates:
            return midpoint(max(candidates, key=duration))

    # 降级轮：扩展整个 [target, maximum] 范围，降低静音阈值到 fallback_silence
    candidates = [sr for sr in silence_ranges
                  if target <= midpoint(sr) - pos <= maximum and duration(sr) >= fallback_silence]
    if candidates:
        return midpoint(max(candidates, key=duration))

    # 硬切
    return pos + maximum
```

**影响范围**：
- `audio_preprocessor.py` — `plan_segments()` 函数重写
- `config.py` — 新增 3 个配置项
- `test_audio_segmentation.py` — 更新已有测试 + 新增渐进搜索场景测试

### 任务二：CLI 和 API 增加 `auto_segment` 用户控制参数

**目标**：让用户可以在任务级别控制是否启用 VAD 切分。

**参数定义**：
- `auto` (默认) — 系统根据时长阈值自动决定
- `on` — 强制启用切分（即使音频 < 10 分钟也做 VAD 分析）
- `off` — 强制关闭切分（整文件直接转写，不做任何切分）

**数据流**：

```
CLI --auto-segment=off
  → TaskItemCreate.options = {"auto_segment": "off", ...}
    → Task.options_json = '{"auto_segment": "off", ...}'
      → task_runner._promote_preprocessing_tasks() 读取 options_json
        → needs_segmentation 判断时加入 auto_segment 策略
```

**改动点**：
1. `schemas/task.py` — `TaskItemCreate.options` 支持 `auto_segment` 字段
2. `api/tasks.py` — `create_tasks()` 透传 options
3. `task_runner.py` — `_promote_preprocessing_tasks()` 读取 `options_json` 中的 `auto_segment`
4. `cli/commands/transcribe.py` — 新增 `--auto-segment` 选项
5. `cli/commands/task.py` — `create` 命令新增 `--auto-segment` 选项
6. `cli/api_client.py` — `create_tasks()` 确保 options 透传

### 任务三：切分前后输出质量对比评估

**目标**：量化评估 VAD 切分对转写质量的影响。

**评估方法**：
1. 使用 3 个长音频文件分别做两组转写：
   - **基准组**：`auto_segment=off`，整文件直接转写
   - **实验组**：`auto_segment=on`，VAD 切分后并行转写再合并
2. 对比维度：
   - **文本完整性**：基准组字数 vs 实验组字数，计算缺失/多余比例
   - **切分边界质量**：切分点附近是否有重复句、断句、漏字
   - **时间戳连续性**：实验组时间戳是否单调递增、无跳变
   - **整体相似度**：使用文本相似度指标（如编辑距离、BLEU）

**输出**：质量评估脚本 + 评估报告
