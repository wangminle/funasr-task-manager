# 项目规范

## 1. 文档命名必须带日期后缀

所有生成的报告、文档、需求或设计类 Markdown 文件，文件名必须以 `-YYYYMMDD` 结尾。

**示例**：`funasr-scheduling-analysis-20260329.md`、`修复迭代开发计划-20260329.md`

## 2. 测试记录按测试类型分目录存放

测试记录存放在 `7-data/outputs/` 下，按测试形式分目录：

```
7-data/outputs/
├── cli/        # CLI 命令行测试记录
├── e2e/        # 端到端测试记录
└── benchmark/  # 性能基准测试记录
```

生成测试记录时，根据测试类型放入对应目录，不要混放。
