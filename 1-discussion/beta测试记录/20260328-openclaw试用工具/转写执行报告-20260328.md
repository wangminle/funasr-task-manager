```bash
  -H "Content-Type: application/json" \
  -d '{"server_id":"funasr-95","name":"FunASR Server 10095","host":"100.116.250.20","port":10095,"protocol_version":"auto","max_concurrency":4}'

# 批量转写命令
python -m cli transcribe \
  "第1集.mp4" "第2集.mp4" ... "第10集.mp4" \
  --language zh \
  --format txt \
  --output-dir ./output \
  --timeout 3600 \
  --poll-interval 10
```

## 四、技术方案说明

### 4.1 系统架构

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   视频文件      │────▶│  ASR Task Mgr   │────▶│  FunASR 集群    │
│  (MP4/WAV/...)  │     │  (FastAPI后端)  │     │  (3台服务器)    │
└─────────────────┘     └─────────────────┘     └─────────────────┘
                               │
                               ▼
                        ┌─────────────────┐
                        │   文字稿输出    │
                        │   (TXT/JSON)    │
                        └─────────────────┘
```

### 4.2 关键技术点

1. **音频提取：** ffmpeg 自动从视频中提取音频轨道
2. **任务调度：** LPT（最长处理时间优先）算法，自动分配到最优服务器
3. **并发控制：** 每台服务器 4 并发槽位，总并发能力 12
4. **进度追踪：** SSE 实时推送进度，CLI 轮询显示
5. **容错机制：** 断路器 + 指数退避重试

### 4.3 使用的 Skill

**参考文件：** `workspace/7-laboratory/funasr-task-manager/6-skills/funasr-task-manager-web-e2e/SKILL.md`

**实际采用方案：**
- 跳过浏览器 E2E 测试方案（Playwright）
- 采用 CLI 工具批量转写方案
- 原因：批量转写任务无需 UI 交互，CLI 效率更高
