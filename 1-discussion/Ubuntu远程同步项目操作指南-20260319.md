# Ubuntu 远程同步 funasr-task-manager 项目操作指南

> 适用场景：在 Ubuntu 系统上通过 GitHub 远程同步本项目，并完成环境搭建与运行。

## 前置条件

- Ubuntu 系统（18.04+）
- 已安装 `gh`（GitHub CLI）
- 已安装 Python 3.11+、Node.js 20+、ffmpeg
- 远程仓库：`https://github.com/wangminle/funasr-task-manager.git`

## 一、确认 gh 登录状态

```bash
gh auth status
```

如果显示未登录，执行登录：

```bash
gh auth login
```

按提示选择：
1. GitHub.com
2. HTTPS 协议
3. 用浏览器或 Personal Access Token 完成认证

## 二、克隆仓库

```bash
gh repo clone wangminle/funasr-task-manager
cd funasr-task-manager
```

## 三、准备测试数据

`7-data/` 目录在 `.gitignore` 中，不会被 Git 同步，需要手动创建并传输测试素材。

### 3.1 创建目录结构

```bash
mkdir -p 7-data/assets/1-测试audioFiles
mkdir -p 7-data/outputs/e2e
```

### 3.2 从 Windows 端传输测试文件

**方式一：在 Ubuntu 端用 rsync 拉取**

```bash
rsync -avz --progress \
  wangminle@<Windows机器IP>:"/mnt/c/Users/wangminle/Documents/VSCode/1-Cursor实战记录/5-FunASR-Practice/funasr-task-manager/7-data/" \
  ./7-data/
```

**方式二：在 Windows 端用 scp 推送**

```powershell
scp -r "7-data\assets" <Ubuntu用户名>@<Ubuntu的IP>:~/funasr-task-manager/7-data/
```

**方式三：通过网盘/U盘等其他方式手动拷贝**

将以下文件放入 `7-data/assets/1-测试audioFiles/` 目录：

| 文件名 | 格式 | 大小 |
|--------|------|------|
| test001.wav | WAV | 3.7 MB |
| tv-report-1.wav | WAV | 5.5 MB |
| test.mp4 | MP4 | 6.4 MB |
| 办公平台20250724-144218.mp4 | MP4 | 9.2 MB |
| tv-report-1.mp4 | MP4 | 20.6 MB |
| 20240510_160113.m4a | M4A | 36.7 MB |
| 241002-GuruMoringTeaching.mp3 | MP3 | 178.9 MB |
| teslaFSD12.x-trial.mp4 | MP4 | 412.6 MB |

## 四、启动后端

```bash
cd 3-dev/src/backend

# 安装依赖
pip install -e ".[dev]"

# 数据库迁移
alembic upgrade head

# 启动服务（监听 0.0.0.0:8000，开启热重载）
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

验证后端就绪：

```bash
curl http://127.0.0.1:8000/health
# 期望返回: {"status":"ok",...}
```

## 五、启动前端

```bash
cd 3-dev/src/frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
# 默认监听 http://localhost:3000 (vite.config.js 配置)
```

或者使用一键脚本（前端会跑在 5173 端口）：

```bash
cd 3-dev/src
bash start.sh
```

## 六、日常代码同步

### 6.1 拉取最新代码（Windows 端推送后）

```bash
git pull origin main
```

### 6.2 推送本地修改（Ubuntu 端开发后）

```bash
git add .
git commit -m "描述信息"
git push origin main
```

### 6.3 查看远程仓库状态

```bash
gh repo view wangminle/funasr-task-manager --web
```

## 七、运行测试

```bash
cd 3-dev/src/backend

# 全量测试 + 覆盖率
python -m pytest "../../../4-tests/scripts/" -v --cov=app

# 仅单元测试
python -m pytest "../../../4-tests/scripts/unit/" -v

# 仅集成测试
python -m pytest "../../../4-tests/scripts/integration/" -v

# E2E 测试
python -m pytest "../../../4-tests/scripts/e2e/" -v
```

## 八、核心原则

| 内容 | 同步方式 | 说明 |
|------|---------|------|
| 源代码 | Git / GitHub | `git pull` / `git push` |
| 测试数据 (`7-data/`) | rsync / scp / 手动 | 在 `.gitignore` 中，不走 Git |
| 运行时数据 (`.runtime/`) | 不同步 | 每台机器本地生成 |
| 数据库文件 (`data/`) | 不同步 | 每台机器通过 `alembic upgrade head` 独立初始化 |
